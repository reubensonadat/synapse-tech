# World 2: Synapse Systems (Product Side)

**Identity:**
- **Tagline:** We don’t just build software, we build systems.
- **Tone:** Visionary, Honest, Long-term, Builder-focused.
- **Visuals:** Deep Black backgrounds, White text, Dark Slate cards. Cinematic and ambitious.

## Navigation Structure
- Our Systems
- Impact
- Roadmap
- Collaborate / Invest
- About (Shared)
- *Switch:* "Need client services instead? → Synapse Studio"

## Content Requirements

### 1. Our Systems Page
**Goal:** Present all platforms built or being built.

**Required Fields for Each System:**
- **Problem:** What real issue exists?
- **Solution:** How the system addresses it.
- **Who It’s For:** Target users or groups.
- **Status:** Live / Beta / Building.
- **Impact:** Users, reach, or value.

**CTAs:**
- "Join the beta"
- "Follow our journey"

*(Note: No specific existing systems were found in `src`. Placeholders or new content will be needed here during development.)*

### 2. Impact Page
**Goal:** Demonstrate that systems are real, used, and growing.
**Focus:**
- Honest metrics.
- Early wins.
- Learning milestones.
- *Builds trust without exaggeration.*

### 3. Roadmap Page
**Goal:** Show direction, not promises.
**Content Categories:**
- What’s building.
- What’s planned.
- What’s being explored.

### 4. Collaborate / Invest Page
**Goal:** Invite aligned people, not random money.
**Audience:** Partners, Early supporters, Contributors.
**CTAs:**
- "Partner with us"
- "Follow our journey"
- *No aggressive fundraising language.*

### 5. Shared DNA (Inherited from Brand Foundation)
*Although "Systems" is a new world, it shares the core values of Synapse:*
- **Innovation:** Pushing boundaries to build platforms.
- **Transparency:** Building systems that are honest and open.
- **Continuous Growth:** Evolving with technology.
